Venge.io JavaScript Modding Guide

A comprehensive overview of common JavaScript modifications for Venge.io.

1. Character and Weapon Customization

Custom Character Skins and Materials

Many mods override the game's default setHeroSkin and createPlayer functions to allow for custom character models (e.g., .glb files) or the application of new materials and textures (e.g., .jpg files) to existing character skins. This can include adding glowing effects or changing default appearances for specific characters like "Shin".

Example: Overriding setHeroSkin for Custom Models/Materials


pc.Application.getApplication().once("postrender", function () {
    pc.app.scripts.list()[11].prototype.setHeroSkin = function() {
        var self = this;
        var skinName = this.heroSkin;

        if (this.heroSkin == 'Default') {
            skinName = this.hero + '-' + this.heroSkin + '.jpg';
        }

        if (skinName && this.characterEntity) {
            if (skinName.search('Model-') > -1 || skinName.search('.glb') > -1) {
                var assetID = this.app.assets.find(skinName).id;
                this.characterEntity.model.asset = assetID;
                // ... (rest of the asset loading logic)
            } else {
                var material = this.characterEntity.model.material.clone();
                var texture = pc.app.assets.find(this.hero + '-' + this.heroSkin + '.jpg');
                // ... (material application and emission effect)
            }
        }
    };
});

Weapon Position Adjustment

JavaScript can be used to set a custom position for the player's weapon, moving it closer, farther, or to a different offset. This is often achieved by modifying the setLocalPosition of the WeaponCenter or weaponHolder entities.

Example: Setting a Fixed Weapon Position


const weaponpos = { x: -0.01, y: 0, z: 0.45 }; // Example coordinates

pc.app.on('Player:Focused', function (state) {
    try {
        if (!state) { // Apply when not focused (e.g., not aiming down sights)
            pc.app.root.findByName("WeaponCenter").setLocalPosition(weaponpos.x, weaponpos.y, weaponpos.z);
        }
    } catch (error) {
        console.log(error);
    }
});

Example: Simple Toggle for Weapon Position


// Weapon position (1 = default, 0 = down, 2 = high)
window.weaponPosition = 0; // Set to 0 for "down" position

Hiding Character Hands/Arms

For a cleaner screen, some mods hide the player's hands or arms by scaling down the CharacterHandsHolder entity or setting a specific hideArms setting.

Example: Hiding Hands


pc.app.on("Map:Loaded", () => {
    setTimeout(() => {
        pc.app.root.findByName("WeaponCenter").findByName("CharacterHandsHolder").setLocalScale(0, 0, 0);
        pc.settings.hideArms = true;
    }, 2000);
});

2. Lighting and Environment Control

Map Lighting Modes (Golden/Dark)

Mods can introduce custom lighting presets like "golden" hour effects or "dark" modes. These modify exposure, skyboxIntensity, and lightColor properties of the game's renderer and scene light.

Example: Applying Golden Lighting Mode


function applyMapLighting(mode) {
    try {
        let exposure = 1.0;
        let skyboxIntensity = 1.5;
        let lightColor = { r: 1, g: 1, b: 1, a: 1 }; // Default white lighting

        if (mode === "golden") {
            exposure = 0.4;
            skyboxIntensity = 2.0;
            lightColor = { r: 1.0, g: 0.85, b: 0.6, a: 1 }; // Golden lighting
        }

        pc.app.renderer.scene.exposure = exposure;
        pc.app.renderer.scene.skyboxIntensity = skyboxIntensity;

        const sceneLight = pc.app.root.findByName("Light");
        if (sceneLight && sceneLight.light) {
            sceneLight.light.color = lightColor;
        }
    } catch (e) {
        console.error("Error applying map lighting:", e);
    }
}

// To activate golden mode:
// applyMapLighting("golden");

Map-Specific Visual Adjustments

For more granular control, mods can apply unique exposure, skybox intensity, and light color settings based on the currently loaded map (e.g., "Sierra", "Mistle", "Xibalba").

Example: Adjusting Lighting for "Sierra" Map


// Inside a Map:Loaded event handler
if (pc.currentMap == "Sierra") {
    pc.app.renderer.scene.exposure = 0.1;
    pc.app.renderer.scene.skyboxIntensity = 10;
    pc.app.root.findByName("Light").light.color = { r: 0.2, g: 0.2, b: 0.2, a: 1 };
}

3. UI Modifications

Hiding UI Elements

Many UI elements can be hidden or disabled to create a cleaner, less cluttered interface. This includes hiding the score background, banners, leaderboard, objective displays, and kill/death feeds.

Example: Hiding Various UI Components


// Inside an initialization or Map:Loaded function
const overlay = pc.app.root.findByName("Overlay");
if (overlay) {
    overlay.findByName("TimesAndScore").findByName("Background").enabled = false;
    overlay.findByName("728x90-Banner").element.opacity = 0;
    // ... many other elements can be disabled or set to opacity 0
    overlay.findByName("ObjectiveHolder").findByName("Objective").enabled = false;
    // Hide leaderboard elements
    pc.app.on("Overlay:Leaderboard", () => {
        try {
            for (let i = 0; i < pc.app.root.findByName("Overlay").findByName("Leaderboard").children.length; i++) {
                pc.app.root.findByName("Overlay").findByName("Leaderboard").children[i].element.opacity = 0.0;
            }
        } catch (error) {}
    });
}

4. Reticle Customization

Dynamic Reticles

Mods can create custom canvas elements to draw dynamic reticles that change based on the equipped weapon, especially when the player is aiming down sights (Player:Focused event). This allows for custom red dot, ACSS, or SMG style reticles.

Example: Drawing Weapon-Specific Reticles


const canvas2 = document.createElement('canvas');
const ctx2 = canvas2.getContext('2d');
canvas2.width = 250;
canvas2.height = 250;
document.body.appendChild(canvas2);
canvas2.style.position = 'absolute';
canvas2.style.top = '50%';
canvas2.style.left = '50%';
canvas2.style.transform = 'translate(-50%, -50%)';

pc.app.on('Player:Focused', (state) => {
    if (state) {
        let currentWeapon = pc.app.root.findByName('Player').script.player.weaponManager.currentWeaponName;
        if (currentWeapon === "Scar" || currentWeapon === "AK-47") {
            drawRedDotReticle();
        } else if (currentWeapon === "Sniper" || currentWeapon === "AWP") {
            drawACSSReticle();
        } else if (currentWeapon === "Tec-9" || currentWeapon === "M4") {
            drawSMGReticle();
        } else {
            console.log(currentWeapon);
        }
    } else {
        clearCanvas(); // Clear when not focused
    }
});

// Example of a drawing function (simplified)
function drawRedDotReticle() {
    ctx2.clearRect(0, 0, canvas2.width, canvas2.height);
    ctx2.beginPath();
    ctx2.arc(canvas2.width / 2, canvas2.height / 2, 2, 0, Math.PI * 2, true);
    ctx2.fillStyle = 'red';
    ctx2.fill();
}

5. Audio Control

Weapon Volume Adjustment

The volume of specific weapons can be adjusted by iterating through weapon entities and setting their sound.volume property.

Example: Setting Weapon Volume


const weaponVolume = 0.5; // Value ranges from 0 to 1

pc.app.on('Map:Loaded', () => {
    const weapons = [
        'Scar', 'Shotgun', 'Sniper', 'Tec-9', 'M4', 'LMG',
        'Desert-Eagle', 'Dagger', 'AK-47', 'AWP'
    ];
    weapons.forEach((weaponName) => {
        const weaponEntity = pc.app.root.findByName(weaponName);
        if (weaponEntity && weaponEntity.sound) {
            weaponEntity.sound.volume = weaponVolume;
        }
    });
});

Character Sound Volume Adjustment

Specific character sounds (like jump or grunt noises) can have their volumes individually adjusted.

Example: Adjusting Character Jump Sound Volume


// Inside a `gameplayStuff` or `Map:Loaded` function
const characterSound = pc.app.root.findByName("Player").script.player.characterEntity.findByName("Sound");
if (characterSound && characterSound.sound) {
    characterSound.sound.slots["Shin-Jump-1"].volume = 0.5;
    characterSound.sound.slots["Lilium-Jump-1"].volume = 0.6;
    // ... and more character sounds
}

Headshot Sound Override

The default headshot sound and visual feedback can be customized by overriding the onHeadshot prototype function, allowing for custom sounds and hitmarker animations.

Example: Custom Headshot Feedback


setTimeout(() => {
    pc.app.scripts.list()[7].prototype.onHeadshot = function (t, e) {
        this.app.fire("DealtDamage:Trigger", e, t, true);
        this.screenEntity.sound.play("Headshot"); // Custom sound
        this.hitmarkerEntity.element.opacity = 1;
        this.hitmarkerEntity.setLocalScale(0.3, 0.3, 0.3);
        this.hitmarkerEntity
            .tween(this.hitmarkerEntity.getLocalScale())
            .to(
                { x: 1.2, y: 1.2, z: 1.2 },
                0.25,
                pc.BackOut
            )
            .start();
        this.lastHeadshot = Date.now();
    };
}, 1000);

6. Gameplay Mechanics and Overrides

Auto-Click Functionality

Some scripts include timed functions that automatically click on specific UI elements, potentially for automated game option selection or similar actions.

Example: Auto-Clicking a Game Option


// After a delay, click the 4th option link
setTimeout(() => {
    document.querySelectorAll('.options a')[3].click();
}, 4000); // Clicks after 4 seconds

Player Leave Redirect

A mod can redirect the player to a different URL (e.g., the Venge.io homepage) when they leave a game session.

Example: Redirect on Player Leave


pc.app.on("Player:Leave", () => {
    window.location.replace("https://venge.io");
});

7. Automated Chat Messages

Sending Random Messages

Mods can be configured to send predefined random messages to the in-game chat when a specific key is pressed (e.g., the "#" key).

Example: Sending a Random Chat Message on Key Press


const messages = [
  "RVNG ON TOP GG EZ!",
  "RVNG EZ WIN!",
  "RVNG IS THE BEST!",
  "EZ LOL RVNG ON TOP!",
  "SMOKED BY RVNG!",
];

function sendRandomMessage() {
  const randomIndex = Math.floor(Math.random() * messages.length);
  const message = messages[randomIndex];
  pc.app.fire("Network:Chat", message); // Send message to game chat
}

function handleKeyPress(event) {
  const key = event.key;
 
  if (key === "#") {
    sendRandomMessage();
  }
}

document.addEventListener("keydown", handleKeyPress);
