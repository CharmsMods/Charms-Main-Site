Understanding Modding Guidelines

It is crucial to understand and adhere to the official modding guidelines to ensure a fair and enjoyable experience for everyone. Violations can lead to serious consequences, including account termination.

Permitted Modifications:

  • Retexturing of Map Objects
  • Retexturing of Characters
  • Retexturing of UI Objects
  • Altering Weapon/Action Audio Files
  • Equitable Visual Adjustment (visual changes that do not provide an unfair advantage)

Prohibited Modifications:

  • Removal/Modification of Map Objects
  • Modification of Character Models
  • Including Dynamic/Redirecting Links within the scripts
  • Altering the Venge Verification/Season Badges
  • Scripts that provide an unfair advantage (e.g., adding .json or .glb files for new models)

Visual & Audio Modding: Retexturing & Sound

This section focuses on how to replace existing game textures (for maps, characters, UI) and audio files. This is a common and permitted way to personalize your game experience without altering core gameplay mechanics.

Setting Up Your Modding Environment

To begin modding, you first need to ensure your game client is installed. Once installed, the game client will recognize a specific folder structure within your computer's Documents directory.

Navigate to your "Documents" folder (e.g., C:\Users\YourName\Documents). Inside, you need to create a new folder named exactly:

Venge Client

Inside this "Venge Client" folder, you will create four more sub-folders:

  • CSS: For custom CSS files that style the game's main menu and UI.
  • Resource Swapper: This is where you'll place your custom textures and audio files.
  • Skin Swapper: Used for character skin modifications (though retexturing is the focus here). This folder is where you place custom textures for the four main characters:
    • Echo-Default.jpg
    • Kulu-Default.jpg
    • Lilium-Default.jpg
    • Shin-Default.jpg
    These are the default texture names for the four playable characters in the game. You can replace these files with your own custom textures to change the appearance of Echo, Kulu, Lilium, and Shin in-game.
  • Userscript: For JavaScript files that modify game logic or visual behaviors.

The Resource Swapper Folder Structure

The Resource Swapper folder has a very specific internal structure that the game client uses to find and apply your custom assets. Inside Resource Swapper, you must create the following path:

Venge Client/Resource Swapper/files/assets/

The assets folder will then contain many sub-folders, each named with a long number. This long number is a unique ID that the game uses to identify a specific asset (like a wall texture or a weapon sound). Inside each of these "long number" folders, there's yet another folder named 1. Finally, your custom asset goes inside that 1 folder.

Finding Asset IDs with Venge Scraper

Most modders use the Venge Scraper website to find the correct asset IDs and file names.

  1. Go to https://d0sob.github.io/vengeScraper/.
  2. Use the search box to type in what you're looking for (e.g., "wall", "gun_sound", "UI_button").
  3. When you click on a result on the website, the URL for the asset (image or MP3) will appear. This URL contains the crucial "long number" (asset ID) you need for your folder name, and the exact file name and extension.
Example URL: https://venge.io/files/assets/30409946/1/Wall-Texture.jpg

From this URL:
- The Asset ID (long number) is: 30409946
- The exact File Name and Extension is: Wall-Texture.jpg

Performing Your First Texture/Audio Mod

Once you have your custom texture or audio file ready, and you've identified the target asset's ID and file name:

  1. Create the full folder path in your Documents: Venge Client/Resource Swapper/files/assets/[Asset_ID]/1/.
  2. Place your custom texture (e.g., a square image, even a transparent PNG) or audio file (e.g., an MP3) inside the 1 folder.
  3. CRITICAL RULE: Your custom file MUST have the exact same file name and extension as the original asset you found on Venge Scraper. For example, if the original is Wall-Texture.jpg, your custom file must also be named Wall-Texture.jpg. If you try to replace a .jpg with a .png, nothing will happen. This applies to all other file types as well (e.g., .mp3 for audio).
  4. Launch the game client to see your changes!

Beyond Textures: Scripting & Styling Visuals

Modding isn't just about replacing images; it's also about manipulating the game's behavior and user interface through code. JavaScript and CSS files play a crucial role in changing lighting, UI elements, and even character animations.

JavaScript for In-Game Visuals

The game uses JavaScript to control various aspects of the in-game environment and UI. By modifying these scripts, you can alter lighting, hide non-essential elements, and adjust visual effects, as long as they comply with the "Equitable Visual Adjustment" guideline. These files would typically be placed in the Userscript folder within your Venge Client directory.

1. Game Settings & Lighting (settings.js)

This script allows for overriding default game functions and adjusting map-specific visual properties. You can change character skin rendering and modify environmental lighting.

// Example: Adjusting map-specific lighting (from settings.js)
// Ensure these adjustments are 'Equitable Visual Adjustments' and don't provide an unfair advantage.
pc.app.on("Map:Loaded", () => {
  setTimeout(() => {
    if (pc.currentMap == "Sierra") {
      pc.app.renderer.scene.exposure = 0.1; // Controls overall brightness
      pc.app.renderer.scene.skyboxIntensity = 10; // Controls skybox brightness
      pc.app.root.findByName("Light").light.color = { r: 0.2, g: 0.2, b: 0.2, a: 1 }; // Adjusts main light color
    }
  }, 2000);
});

// Example: Character skin emission effect (from settings.js)
// This can make character skins glow. Ensure it's an equitable visual adjustment.
pc.app.scripts.list()[11].prototype.setHeroSkin = function(){
  /* ... (code omitted for brevity) ... */
  material.emissive = new pc.Color(1, 1, 1); // Sets emission color to white
  material.emissiveIntensity = 10000; // Sets emission intensity
  material.update();
  /* ... */
};

The exposure, skyboxIntensity, and light.color properties control the overall brightness, skybox contribution, and color of the main light source, respectively. The emissive and emissiveIntensity properties can make character skins glow. Always ensure these changes do not create an unfair advantage by making players or objects unnaturally visible.

2. UI Hiding & Placement (hide_ui.js)

This script demonstrates how to disable or reposition various in-game UI elements. You can hide non-essential elements like ads or reposition information displays.

// Example: Hiding non-essential UI elements (from settings.js, similar to hide_ui.js)
// Hiding ads and banners is generally permitted as it doesn't provide an unfair advantage.
pc.app.on("Map:Loaded", () => {
  setTimeout(() => {
    pc.app.root.findByName("Overlay").findByName("Announce").setLocalPosition(0, -530, 0); // Reposition an announcement
    pc.app.root.findByName("Achievement").children[0].enabled = false; // Disable an achievement notification
    pc.app.root.findByName("Overlay").findByName("728x90-Banner").element.opacity = 0; // Hide a banner ad
  }, 2000);
});

By setting enabled = false or element.opacity = 0, you can make UI elements disappear. setLocalPosition allows you to move them around the screen.

Prohibited: Hiding Critical Game Information! Hiding UI elements that provide crucial game information (e.g., player health bars, objective markers, full leaderboard details that impact gameplay) is considered to provide an unfair advantage and is strictly prohibited.

3. Weapon Visuals & Audio (weapon_pos.js, weapon_vol.js, hide_hands.js)

These smaller scripts specifically target weapon appearance and sound.

// Example: Adjusting weapon position (from weapon_pos.js)
const weaponpos = { x: -0.01, y: 0, z: 0.45 } // Adjust X, Y, Z coordinates
pc.app.on('Player:Focused', function (state) {
  if (!state) {
    pc.app.root.findByName("WeaponCenter").setLocalPosition(weaponpos.x, weaponpos.y, weaponpos.z)
  }
});

// Example: Hiding player hands (from hide_hands.js)
pc.app.on("Map:Loaded", () => {
  setTimeout(() => {
    pc.app.root.findByName("WeaponCenter").findByName("CharacterHandsHolder").setLocalScale(0, 0, 0); // Sets scale to zero to hide
    pc.settings.hideArms = true // Sets a game setting to hide arms
  }, 2000);
});

// Example: Setting weapon volume (from weapon_vol.js)
// Altering weapon audio is a Permitted Modification.
const weaponVolume = 0.5; // Value ranges from 0 to 1
pc.app.on('Map:Loaded', () => {
  pc.app.root.findByName('Scar').sound.volume = weaponVolume;
  // Apply to other weapons as needed
});

setLocalPosition on WeaponCenter changes where your weapon appears on screen. setLocalScale(0, 0, 0) effectively hides the player's hands. sound.volume directly controls the audio level of specific game objects. These are examples of "Equitable Visual Adjustments" that can be used for competitive purposes without violating core rules.

4. Chat Messages (message.js)

This script demonstrates how to inject custom, pre-defined messages into the in-game chat. This is generally permissible as it does not offer an unfair advantage.

// Example: Sending a random chat message (from message.js)
const messages = [
  "RVNG ON TOP GG EZ!",
  "RVNG EZ WIN!",
];
function sendRandomMessage() {
  const randomIndex = Math.floor(Math.random() * messages.length);
  const message = messages[randomIndex];
  pc.app.fire("Network:Chat", message); // Fires an event to send the message to chat
}
document.addEventListener("keydown", handleKeyPress); // Triggers on key press

This shows how to interact with the game's internal events, in this case, firing a "Network:Chat" event to send a message.

CSS for Main Menu & UI Styling

CSS files are used to completely overhaul the appearance of the game's main menu, popups, and other UI elements. You can change colors, fonts, sizes, and even hide default elements, provided these are "Retexturing of UI Objects" and do not alter verification badges or provide an unfair advantage. These files would typically be placed in the CSS folder within your Venge Client directory.

1. Grey Mod Styling (Grey.css)

This CSS file provides a black and white theme with red accents. It heavily modifies the main menu's navigation, buttons, and information displays.
